# Sprint 3 — Feature Complete
**COP 2250 — Terminal Game Project** | Pyatt Labs @ SCF

---

## Sprint 3 Requirements

By the end of Week 15 your game must be feature complete.

| Requirement | Description |
|---|---|
| All invalid input handled | No crashes from bad input — exception handling required |
| At least one interface implemented | A class implements a defined interface |
| Win/lose/end condition functional | The game can end in a defined outcome |
| Code organized and commented | Every class has a header comment. Every method has a purpose. |
| All OOP requirements met | See project overview for full list |

---

## Playtest Day

On Thursday of Week 15, teams swap games.

**Protocol:**
1. Push your code to your HedgeDoc or share via terminal
2. Another team clones or copies your files
3. They compile and run your game
4. They play for 10 minutes
5. They write 3-5 sentences of feedback

**Feedback must include:**
- What worked well
- What was confusing
- One specific suggestion for improvement

---

## Sprint 3 Demo

Each team member must be able to:
1. Screen-share terminal
2. Compile and run the game live
3. Demonstrate the end condition
4. Explain one class they personally wrote

---

## Checklist Before Playtest
```
[ ] javac *.java compiles with no errors
[ ] java Main (or equivalent) launches the game
[ ] Player can complete at least one full game loop
[ ] Invalid input does not crash the game
[ ] End condition triggers correctly
[ ] All team members can explain their assigned classes
```
